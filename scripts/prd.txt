# Product Requirements Document (PRD)

## 1 Overview

**Project Name**: **holis.social**
**Sponsoring Team**: Verse / Planetary / Freedom Tech crew
**Vision**: Offer activists and conference‑goers a privacy‑respecting, lightning‑fast Progressive Web App (PWA) that makes it effortless to discover people, collaborate in semi‑private groups, and move money (donations, tips, pooled funds) with Cashu—all without ever mentioning “Nostr” or “Bitcoin” up‑front. We will build an MVP in **7 days** and unveil it at the **Oslo Freedom Forum 2025** Freedom Tech track on **28 May 2025**.

| Goal                     | KPI / Target                                          |

## 5 User Journeys

### 5.1 Onboarding & Contact Creation (Day‑0)

1. Attendee scans lobby poster QR ➜ PWA opens.
2. Welcome splash explains purpose *in plain language*; tap **Start**.
3. App autogenerates keypair, pseudonym (e.g., “Rambunctious Sloth”), and Cashu wallet.
4. User optionally edits display name & tags (#disability, #climate…) and taps **Done**.
5. Home screen shows **Public Square** + prompt to *“Join or create a group”*.

### 5.2 Secure Contact Exchange

* In person: each phone shows rotating QR of their Contact Card; scanning adds to Rolodex.
* Remote: share short ‘holis.social/@alias’ link.
* Contacts searchable offline; notes private‑only.

### 5.3 Private Group Flow

* Create ➜ set name & description ➜ get invite link/QR ➜ members join ➜ chat (kind:4) appears.
* Admins can pin **Donate** button tied to group wallet.

### 5.4 Panic Mode

* **Danger Zone** in Settings ➜ *“Nuke everything”* ➜ confirm ➜ secure erase; app returns to welcome.

## 6 Security & Privacy Requirements

| Concern               | Requirement                                         | Rationale                                                                                                                                                    |
| --------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Metadata minimization | Store only relay URLs & npub locally; no central DB | Activist concerns (Eichenmüller 2025) ([journals.sagepub.com](https://journals.sagepub.com/doi/pdf/10.1177/2053951720985557?utm_source=chatgpt.com))         |
| Location safety       | Allow delayed send + local‑only drafts              | Ambush prevention (Eichenmüller 2025) ([journals.sagepub.com](https://journals.sagepub.com/doi/pdf/10.1177/2053951720985557?utm_source=chatgpt.com))         |
| E2E indicator         | Persistent lock icon & key‑rollover animation       | Visibility improves trust (Eichenmüller 2025) ([journals.sagepub.com](https://journals.sagepub.com/doi/pdf/10.1177/2053951720985557?utm_source=chatgpt.com)) |
| Rapid data burn       | Panic Mode wipes IndexedDB, LocalStorage, Cache     | “Anticipatory data practices” ([journals.sagepub.com](https://journals.sagepub.com/doi/pdf/10.1177/2053951720985557?utm_source=chatgpt.com))                 |
| Abuse mitigation      | Inline flag/report ➜ mod queue                      | Platform violence data (El‑Masri 2024) ([journals.sagepub.com](https://journals.sagepub.com/doi/pdf/10.1177/14614448241248372?utm_source=chatgpt.com))       |

## 7 Technical Architecture

* **Frontend**: React 18 + Vite, TailwindCSS 3 ([blog.rsroshi.dev](https://blog.rsroshi.dev/why-tailwind-in-2025-the-best-css-framework-for-modern-web-development/?utm_source=chatgpt.com))
* **PWA layer**: Service‑worker caching; offline‑ready (Flexxited guide) ([flexxited.com](https://flexxited.com/blog/the-ultimate-guide-to-progressive-web-apps-pwas-in-2025-benefits-and-best?utm_source=chatgpt.com))
* **Cryptography**: nostr‑tools for keys; secp256k1 wasm.
* **Cashu**: `cashu-ts` SDK for wallet ops ([github.com](https://github.com/cashubtc/cashu-ts?utm_source=chatgpt.com)); mints configured via NIP‑60 tags.
* **Backend**: Custom relay for group & media; Deno deploy.
* **CI/CD**: GitHub Actions ➜ Vercel preview ➜ production.
